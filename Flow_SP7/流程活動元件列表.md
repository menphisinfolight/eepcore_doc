## 元件繼承架構

```
Activity（基底，1125 行）
├── ContinuedActivity
│   ├── StandActivity        — 標準簽核
│   ├── ApproveActivity      — 主管簽核
│   ├── MultiApproveActivity — 多人會簽
│   ├── SeqApproveActivity   — 循序會簽
│   ├── NotifyActivity       — 知會通知
│   ├── CommentActivity      — 評論
│   ├── ReturnActivity       — 退回
│   └── RejectActivity       — 作廢
├── DetailFlowActivity       — 明細流程
├── ParallelActivity         — 並行
├── IfElseActivity           — 條件判斷
├── ValidateActivity         — 驗證
├── ProcedureActivity        — 程序
├── SubFlowActivity          — 子流程
├── WarningActivity          — 預警
├── RootActivity             — 根活動
└── ExceptionActivity        — 例外
```

## 流程狀態流轉

```
Start（啟動）
  → RootActivity
  → StandActivity / ApproveActivity / ...
  → Submit（送出）→ 下一個活動
  → Return（退回）→ 指定活動
  → Reject（作廢）→ 流程結束
  → End（結案）→ 流程完成
```
